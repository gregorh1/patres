## Project Overview

Patres is a Flutter mobile app for reading Church Fathers and Christian Classics texts. It's an offline-first reader with SQLite-backed FTS5 full-text search, reading plans, bookmarks, highlights, and TTS audiobook generation. Supports Polish and English localization.

## Common Commands

```bash
flutter pub get              # Install dependencies
flutter analyze              # Lint (uses flutter_lints)
flutter test                 # Run all tests
flutter test test/blocs/     # Run BLoC tests only
flutter test test/services/  # Run service tests only
flutter test test/models/    # Run model tests only
flutter test test/blocs/reader_bloc_test.dart  # Run a single test file
flutter run                  # Run on connected device
flutter build apk            # Build Android APK
```

## Architecture

**BLoC pattern** with clean layer separation: Screens -> BLoCs -> Services -> Database/Assets.

### Layer responsibilities

- **`lib/screens/`** — Stateless/stateful widgets that consume BLoC state. No business logic.
- **`lib/blocs/`** — State management via flutter_bloc. Each BLoC has separate `_event.dart` and `_state.dart` files. Events trigger state emissions.
- **`lib/services/`** — Data access layer. Services abstract SQLite, asset loading, and external APIs (TTS). Injected into BLoCs via constructor.
- **`lib/models/`** — Immutable data classes using Equatable. JSON serialization where needed.
- **`lib/widgets/`** — Reusable UI components shared across screens.

### Navigation

GoRouter (`lib/router.dart`) with two navigator keys:
- **Root navigator** — full-screen routes: `/splash`, `/reader/:id`, `/search`, `/audio-downloads`, `/plans/:id`, `/author/:id`
- **Shell navigator** — tab-based main nav via `ShellScaffold`: `/home`, `/library`, `/plans`, `/settings`

BLoCs for reader, search, and TTS are scoped per-route in the router (created in route builders), not provided globally.

### Database

`DatabaseService` (`lib/services/database_service.dart`) manages SQLite via sqflite with FFI backend (for FTS5 support). Database version 3 with migrations. Tables: `reading_progress`, `scroll_positions`, `bookmarks`, `highlights`, `search_fts` (FTS5 virtual table), `plan_progress`, `plan_day_completions`.

FFI initialization happens in `main()` — `sqfliteFfiInit()` + `databaseFactory = databaseFactoryFfi` — to ensure the bundled SQLite (with FTS5) is used instead of Android's framework SQLite.

### Theming

Three themes (light/dark/sepia) in `lib/theme.dart` via `PatresTheme.themeFor(AppThemeMode)`. Material 3 with a burgundy/gold/parchment brand palette. Theme state managed by `ThemeBloc`.

### Localization

ARB-based (`lib/l10n/app_pl.arb`, `app_en.arb`). Polish is the template language. Config in `l10n.yaml`. Generated code in `lib/l10n/generated/` — auto-generated by Flutter (`flutter: generate: true` in pubspec).

### Content

Texts are bundled as JSON assets in `assets/texts/`. A `manifest.json` lists all available texts. Additional assets: `authors.json`, `daily_readings.json`, `reading_plans.json`.

## Testing Patterns

Tests use `bloc_test` for BLoC testing and `flutter_test` for services/models. Services are tested with in-memory fakes (e.g., `FakeDatabaseService`, `FakeTextService`) — no mocking framework. Database tests use `sqflite_common_ffi` with `inMemoryDatabasePath` for real SQLite in tests.

## Key Dependencies

- **State**: flutter_bloc, equatable
- **Navigation**: go_router
- **Database**: sqflite + sqflite_common_ffi + sqlite3_flutter_libs (FTS5)
- **Audio**: just_audio, audio_service
- **HTTP**: http (for TTS API via `TtsAudioService`)
- **SDK**: Dart ^3.11.1
